## 3. 提案方針（結論）

- **まずは「顧客管理をしっかりできるところまで」を目標にします。**  
  名刺・顧客マスタ・接点（活動・対応）の一元化と、**タスク管理まで** を一つの区切りとし、「誰が何をしているか」が見える状態を目指します。

- **案件管理・契約管理・売上管理・会計連携** は、kintone への入力に慣れた**あと**に実施する想定です。土台を固めてから追加することで、定着しやすくし、投資の段階化も図ります。

- 社内にシステム専任部門がなく、**ノーコード・低コスト** での改善を重視されている前提で設計します。

## 4. 現状の課題（打合せでお伺いした内容）

複数部門・複数法人で運営されるなか、顧客や接点の情報が部門単位で分散しており、グループ全体で「誰がどこを担当しているか」「どこに売上があるか」を把握しづらい状況とお聞きしています。

| 分類 | 課題 |
| ------ | ------ |
| **情報の分断** | 各部門が独立運営のため、顧客・営業情報が部門ごとにバラバラで、横串で検索・参照できない。 |
| **ツールの混在** | 顧客管理は Excel・一部専用システム等が混在。名刺も部門ごとに個別管理で一元化されていない。 |
| **属人化** | 接点履歴が個人に依存し、グループ全体で「誰がどこと接点を持っているか」が見えない。担当者がいないと情報がたどれない。 |
| **重複・機会損失** | 他部門ですでに訪問している企業に気づかず訪問するなど、営業資産が活かしきれていない。 |
| **既存ツール** | マネーフォワード、rakumo、Eight Team 等が点在。将来的な連携を考慮したいが、現状は手作業になりがち。 |

## 5. 解決策（ご提案の方向性）

| 課題 | 解決策（kintone で実現するイメージ） |
| ------ | ------ |
| 情報の分断 | 法人番号をキーにした顧客（法人）ベースのデータベースを kintone 上で整備し、グループ横断で一覧・検索できるようにする。 |
| ツールの混在 | 顧客マスタを kintone で構築。名刺は Eight 連携 or CSV で取込めるようにし、顧客管理・名刺を一元化する。 |
| 属人化 | 顧客に紐づいた活動・問い合わせ・クレームを一元管理し、部門長も確認できる形で蓄積。「誰がどこと接点を持っているか」をグループ全体で見えるようにする。 |
| 重複・機会損失 | 接点・活動の一元管理で「どこに誰が訪問したか」が分かる形にする。売上・会計連携は次のフェーズで検討する。 |

## 6. なぜ2段階で進めるか

| 観点 | 説明 |
| ------ | ------ |
| **土台の整備** | 案件・売上はすべて「どの顧客か」に紐づく。まず顧客マスタと接点を整え、顧客単位で情報が蓄積される状態にしてから、案件・売上・会計を紐づける方が二度手間がない。 |
| **習慣づくり** | 複数部門・約60名が対象のため、Phase1 で「顧客と接点を記録する」習慣をつけていただき、そのうえで Phase2 を追加する方が定着しやすく、リスクも分散できる。 |
| **投資の段階化** | Phase1 で効果を確認したうえで Phase2 の範囲と予算を決めていただく形にし、負担と成果のバランスを取りやすくする。 |

**Phase の整理**

| Phase | 内容 |
| ------ | ------ |
| **Phase1** | 顧客管理をしっかり（**タスク管理まで**）。名刺・顧客マスタ・接点の一元化とタスク管理・日報を構築。「誰が何をしているか」が見える状態を目指す。 |
| **Phase2** | kintone に慣れた**あと**に実施。案件管理・契約管理・売上管理の構築と、マネーフォワード等との会計連携を検討する。 |

## 7. システム構成イメージ

### 7-1. データの流れ（Phase1 の範囲）

```
［名刺：Eight Team 等］
        ↓ 取込（連携 or CSV）
［法人番号ベースの顧客マスタ］  … 1企業1キー
        ＋
［活動管理］［対応管理］  … 顧客に紐づけて蓄積
        ＋
［タスク管理］  … 部門ごとにアプリを分けて運用
        ＋
［日報］  … タスクと紐づけて参照
        ↓ Phase2（慣れたあと）
［案件管理］［契約管理］［売上管理］  … 顧客に紐づけ。会計連携は Phase2 で検討
```

- **法人番号** を共通キーにし、名刺・顧客マスタ・活動・対応・タスク・日報を横串で参照できる形を目指します。
- kintone 上ではアプリ同士を関連レコード等で紐付け、顧客画面から活動・対応・タスク・日報を参照できるようにします。

### 7-2. 主要な機能（Phase1）

| 機能 | 内容 |
| ------ | ------ |
| 名刺・顧客マスタ | Eight 連携 or CSV 取込。法人番号を軸にした顧客マスタ。部門ごとに参照範囲を切り分け。 |
| 活動管理 | 訪問・電話・メール・商談メモなど「誰が・いつ・何をしたか」を記録。顧客に紐づけて蓄積。 |
| 対応管理 | 問い合わせ・クレームを種別で管理。対応期限・担当・ステータス・内容を一元管理。 |
| タスク管理 | 「いつまでに何を完了するか」を管理。部門ごとにアプリを分け（例：税理士法人用・社労士法人用）、混在を防ぐ。 |
| 権限設計 | 複数法人・部門を考慮したアクセス権（グループや組織で見える範囲を制御）。 |

### 7-3. 既存ツールとの関係

- **Eight Team**：名刺の取込起点。kintone 連携 or CSV で顧客マスタに反映する想定。
- **マネーフォワード**：Phase2 で会計連携を検討。API の利用可否等は要確認。
- その他（rakumo 等）：必要に応じ、手作業や CSV で「箱」に格納するイメージ。

## 8. フェーズとスコープ

### 8-1. Phase1 で目指す状態

- 法人番号ベースでグループ横断の顧客・営業情報を一覧できる。
- 名刺を一元管理（Eight 連携 or CSV）。
- 顧客に紐づいた部門別情報・活動・問い合わせ・クレームを蓄積し、部門長も確認できる。
- 部門別タスク管理・日報連携・表示で「誰が何をしているか」が見える。
- 権限設計を反映した運用。データ移行・研修まで完了し、運用開始できる状態。

### 8-2. Phase2 で目指す状態（慣れたあとに実施）

- 案件管理・契約管理・売上管理アプリを顧客に紐づけて構築。
- マネーフォワード等との会計連携（API 可否は要確認）。
- テスト・納品・研修まで完了し、運用開始できる状態。

### 8-3. スコープの境界

- **Phase1 に含める**：要件定義・設計、名刺連携（Eight 想定）、顧客マスタ・部門別情報・活動管理・対応管理、タスク管理（部門別分割）、日報連携・表示、権限設計、初期データ整備支援、テスト・納品・研修。
- **Phase2 に含める**：案件・契約・売上管理アプリの構築、会計連携（マネーフォワード等）の設計・実装（要確認）、テスト・納品・研修。
- **保守・運用**：軽微な改修は別途契約。運用全体の整備も別途相談。
